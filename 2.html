Building reusable UI building blocks

Understanding React Components

Components are the heart of React applications. They are independent, reusable pieces of code that return JSX to describe what should appear on the screen.

What Makes a Component?

🔄
Reusable: Write once, use everywhere
🧩
Composable: Build complex UIs from simple pieces
🔧
Maintainable: Easy to update and debug
⚡
Performant: React optimizes rendering automatically

💡 Component Philosophy

Think of components like LEGO blocks. Each block has a specific purpose and can be combined with others to build amazing structures.

2

Function Components vs Class Components

React supports two types of components. Modern React development primarily uses function components with hooks.

Function Components (Recommended)

Modern Function Component

import React from 'react';

function Welcome(props) {
  return (
    <div>
      <h1>Hello, {props.name}!</h1>
      <p>Welcome to React</p>
    </div>
  );
}

export default Welcome;

Class Components (Legacy)

Class Component Example

import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return (
      <div>
        <h1>Hello, {this.props.name}!</h1>
        <p>Welcome to React</p>
      </div>
    );
  }
}

export default Welcome;
Function Components Class Components
✅ Modern, preferred approach ⚠️ Legacy, being phased out
✅ Use hooks for state & effects ✅ Use lifecycle methods
✅ Less code, easier to read ❌ More boilerplate code
✅ Better performance ❌ Slightly slower

JSX: The Language of React

JSX is a syntax extension that allows you to write HTML-like code in JavaScript. It's not required but makes React code much more readable.

JSX Basics

Basic JSX

const element = <h1>Hello, world!</h1>;

Simple JSX element

Embedding Expressions

const name = "React";
const element = <h1>Hello, {name}!</h1>;

Use curly braces for JavaScript expressions

Attributes

const element = <img src={user.avatarUrl} />;

Dynamic attributes with expressions

Children

const element = (
  <div>
    <h1>Title</h1>
    <p>Content</p>
  </div>
);

JSX elements can have children

JSX Rules & Best Practices

1
Always close tags: Use self-closing tags for elements without children
2
One parent element: JSX expressions must have one parent element
3
className not class: Use className instead of class for CSS classes
4
htmlFor not for: Use htmlFor instead of for for label elements

Props: Passing Data to Components

Props (properties) are how you pass data from parent components to child components. They are read-only and help make components reusable.

Basic Props Usage

Component with Props

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Usage in parent component
function App() {
  return (
    <div>
      <Greeting name="Alice" />
      <Greeting name="Bob" />
      <Greeting name="Charlie" />
    </div>
  );
}

Destructuring Props

Cleaner Props with Destructuring

// Instead of: function User(props) { return <div>{props.name}</div>; }

// Use destructuring:
function User({ name, age, email }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <p>Email: {email}</p>
    </div>
  );
}

// Usage:
<User name="John" age={25} email="john@example.com" />

Default Props

Setting Default Values

function Button({ text, color = "blue" }) {
  return <button style={{ backgroundColor: color }}>{text}</button>;
}

// These are equivalent:
<Button text="Click me" />  // color = "blue"
<Button text="Click me" color="red" />  // color = "red"

Common Prop Types

  • string - Text content
  • number - Numeric values
  • boolean - True/false values
  • function - Event handlers
  • object - Complex data
  • array - Lists of data

State Management with useState

State allows components to manage their own data that can change over time. The useState hook is the primary way to add state to function components.

Basic useState Usage

Counter Component

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Current count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
      <button onClick={() => setCount(count - 1)}>
        Decrement
      </button>
    </div>
  );
}

State with Different Data Types

String State

const [name, setName] = useState("John");

Number State

const [age, setAge] = useState(25);

Boolean State

const [isVisible, setIsVisible] = useState(true);

Object State

const [user, setUser] = useState({ name: "John", age: 25 });

Array State

const [items, setItems] = useState(["item1", "item2"]);

Updating Object/Array State

Updating Objects

const [user, setUser] = useState({ name: "John", age: 25 });

// ❌ Wrong way (mutates state directly)
setUser({ name: "Jane" }); // Loses age property

// ✅ Correct way (uses spread operator)
setUser({ ...user, name: "Jane" }); // Preserves age

// Or use functional update
setUser(prevUser => ({ ...prevUser, name: "Jane" }));

State Update Rules

  • ✅ State updates are asynchronous
  • ✅ Never modify state directly
  • ✅ Use functional updates for previous state
  • ✅ State can trigger re-renders
  • ✅ Multiple setState calls may batch together

Event Handling in React

React provides a consistent way to handle events across different browsers. Event handlers are functions that get called when specific events occur.

Basic Event Handling

Click Event Example

function Button() {
  function handleClick() {
    alert('Button was clicked!');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

Event Object

Accessing Event Information

function handleSubmit(event) {
  event.preventDefault(); // Prevents page reload
  console.log('Form submitted');
}

function handleChange(event) {
  console.log('Input value:', event.target.value);
}

Common Event Types

onClick Fires when element is clicked
onChange Fires when input value changes
onSubmit Fires when form is submitted
onMouseEnter Fires when mouse enters element
onMouseLeave Fires when mouse leaves element
onFocus Fires when element gains focus
onBlur Fires when element loses focus

Passing Arguments to Event Handlers

Event Handlers with Parameters

function handleDelete(id) {
  console.log('Deleting item:', id);
}

function ItemList() {
  const items = [1, 2, 3];

  return (
    <ul>
      {items.map(item => (
        <li key={item}>
          Item {item}
          <button onClick={() => handleDelete(item)}>
            Delete
          </button>
        </li>
      ))}
    </ul>
  );
}

Conditional Rendering

Conditional rendering allows you to show different content based on certain conditions. React supports several patterns for conditional rendering.

If/Else Statements

Using if/else

function Greeting(props) {
  if (props.isLoggedIn) {
    return <h1>Welcome back!</h1>;
  } else {
    return <h1>Please sign in.</h1>;
  }
}

Ternary Operator

Inline Conditional Rendering

function Greeting(props) {
  return (
    <div>
      {props.isLoggedIn ? (
        <h1>Welcome back!</h1>
      ) : (
        <h1>Please sign in.</h1>
      )}
    </div>
  );
}

Logical && Operator

Rendering with &&

function Mailbox(props) {
  return (
    <div>
      <h1>Hello!</h1>
      {props.unreadMessages.length > 0 && (
        <p>
          You have {props.unreadMessages.length} unread messages.
        </p>
      )}
    </div>
  );
}

Preventing Component Rendering

Returning null

function WarningBanner(props) {
  if (!props.warn) {
    return null; // Nothing renders
  }

  return (
    <div className="warning">
      Warning!
    </div>
  );
}

Conditional Rendering Tips

  • Use ternary operators for simple conditions
  • Use && for "if exists" scenarios
  • Return null to hide components completely
  • Extract complex conditions to variables for readability

Lists and Keys

Rendering lists of data is a common pattern in React. Keys help React identify which items have changed, are added, or removed.

Basic List Rendering

Rendering an Array of Items

function NumberList(props) {
  const numbers = props.numbers;
  const listItems = numbers.map((number) =>
    <li key={number.toString()}>
      {number}
    </li>
  );

  return <ul>{listItems}</ul>;
}

// Usage:
const numbers = [1, 2, 3, 4, 5];
<NumberList numbers={numbers} />

Why Keys Are Important

🔑
Identification: Keys help React identify which items have changed
⚡
Performance: Keys enable efficient re-rendering of lists
🎯
Stability: Keys maintain component state during reorders

Choosing the Right Key

✅ Good Keys

  • key={item.id} - Unique database ID
  • key={item.uuid} - Universally unique identifier
  • key={index} - Last resort for static lists

❌ Bad Keys

  • key={Math.random()} - Changes every render
  • key={new Date()} - Not stable
  • No key at all - React warning

Keys in Practice

Todo List with Keys

function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo) => (
        <li key={todo.id}>
          {todo.text}
        </li>
      ))}
    </ul>
  );
}

const todos = [
  { id: 1, text: 'Learn React' },
  { id: 2, text: 'Build an app' },
  { id: 3, text: 'Deploy to production' }
];

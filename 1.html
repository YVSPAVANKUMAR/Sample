⚙️ Setup & Basics

Get your React development environment ready

1

Prerequisites Check

Before diving into React, let's ensure your development environment is properly set up. React requires Node.js and npm (Node Package Manager).

Step 1.1: Check if Node.js is installed

Open your terminal and run:

node --version

If you see a version number (like v18.17.0), Node.js is installed. If not, proceed to the next step.

Step 1.2: Download and Install Node.js

🌐
Visit the official website: https://nodejs.org
📦
Download the LTS version (recommended for most users)
⚙️
Run the installer and follow the setup wizard
✅
Verify installation: node --version && npm --version

💡 Why Node.js?

Node.js allows you to run JavaScript outside of the browser. React applications are built using Node.js tools and npm packages.

3

Create Your First React Application

Now let's create your first React application using Create React App, the official React build tool.

Step 3.1: Open Terminal/Command Prompt

  • Windows: Press Win + R, type "cmd", press Enter
  • macOS: Press Cmd + Space, type "Terminal", press Enter
  • Linux: Press Ctrl + Alt + T

Step 3.2: Navigate to Your Projects Folder

Create a projects directory (if you don't have one):

mkdir react-projects
cd react-projects

Step 3.3: Create React App

Run the create-react-app command:

npx create-react-app my-first-react-app

This command will:

  • Create a new folder called "my-first-react-app"
  • Download all necessary dependencies
  • Set up the project structure
  • Configure build tools

Step 3.4: Navigate into Your Project

Change directory to your new project:

cd my-first-react-app

Step 3.5: Start the Development Server

Launch your React app:

npm start

This will start the development server and open your app in the browser at http://localhost:3000

🎉 Congratulations!

You've successfully created and started your first React application! You should see the default React welcome page in your browser.

4

Understanding Project Structure

Let's explore the files and folders that Create React App generated for you.

📁 my-first-react-app/

Root project directory

📁 src/

Source code directory (where you'll write your React code)

⚛️ App.js - Main application component
🏠 index.js - Application entry point
🎨 App.css - Component styles
🖼️ logo.svg - React logo
🌐 public/

Static assets and HTML template

📄 index.html - Main HTML file
🎯 favicon.ico - Browser tab icon
📋 package.json

Project configuration and dependencies

🔒 package-lock.json

Exact dependency versions (auto-generated)

📖 README.md

Project documentation

Key Files to Remember:

  • src/App.js - Your main component
  • src/index.js - App startup
  • public/index.html - HTML template
  • package.json - Project config
5

Your First Code Changes

Let's make our first modification to the React app to see how changes work.

Step 5.1: Open the Project in VS Code

In your terminal, from the project directory:

code .

This opens the current directory in VS Code.

Step 5.2: Edit App.js

Open src/App.js in VS Code and replace the content with:

src/App.js

function App() {
  return (
    <div className="App">
      <header className="App-header">
        <h1>Hello, React World!</h1>
        <p>My first React application</p>
        <button onClick={() => alert('Hello!')}>
          Click me!
        </button>
      </header>
    </div>
  );
}

export default App;

Step 5.3: Save and See Changes

Save the file (Ctrl+S or Cmd+S). The browser should automatically refresh and show your changes!

🔄 Hot Module Replacement

Create React App includes Hot Module Replacement (HMR), which automatically updates your app in the browser when you save changes to your code.

6

Essential npm Commands

Learn the most important npm commands you'll use regularly in React development.

npm start

Starts the development server

npm start

npm run build

Creates production build

npm run build

npm test

Runs test suite

npm test

npm install

Installs dependencies

npm install package-name

npm run eject

Ejects from CRA (irreversible)

npm run eject

npm run deploy

Deploy to GitHub Pages

npm run deploy

⚠️ Important Warning

npm run eject is irreversible. Only use it if you need full control over build configuration. Most projects never need to eject.
